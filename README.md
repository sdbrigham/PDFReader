# PDF Web Reader

A local PDF reader that lets you open one PDF, scroll it like a normal document, highlight text, and press `Cmd+K` or `Ctrl+K` to ask about the selection.

Answers are generated by a configurable model provider and the server performs a live web lookup for every question. The highlighted PDF text stays the baseline context, while web snippets provide outside background and source links.

## Run

```sh
npm install
npm run dev
```

Open `http://127.0.0.1:5173`.

## Model Provider

The default provider is `auto`. In `auto` mode, the app uses a GPT-oss-compatible endpoint when `MODEL_BASE_URL` or `GPT_OSS_BASE_URL` is configured. Otherwise it uses a reliable grounded PDF-plus-web answer mode so the app still works without a model server.

For a GPT-oss deployment exposed through an OpenAI-compatible `/v1/chat/completions` API:

```sh
MODEL_PROVIDER=gpt-oss \
MODEL_BASE_URL=http://127.0.0.1:8000/v1 \
MODEL_NAME=gpt-oss \
npm run dev
```

Optional:

```sh
MODEL_API_KEY=...
```

Legacy names also work: `GPT_OSS_BASE_URL`, `GPT_OSS_MODEL`, and `GPT_OSS_API_KEY`.

For Vercel, configure those values in project environment variables. The shared answer engine lives in `lib/answerEngine.mjs`, and both the local Express server and the Vercel-style API route use it.

## Local Transformers

You can still run a local Transformers.js-compatible open-source text-to-text model explicitly:

```sh
MODEL_PROVIDER=local-transformers \
LOCAL_OSS_MODEL=Xenova/flan-t5-base \
npm run dev
```

The first local model question may take longer because the model has to download and cache locally. The app quality-gates local model output and falls back to grounded PDF-plus-web answers when generation is poor.
